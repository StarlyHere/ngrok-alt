Developer runbook

Run a QA6 downstream microservice locally

Keep the WebUI request in QA6, intercept one caller-side tier, and carry its raw gRPC connection through the existing session-aware SSH tunnel to your laptop.

Before you start

QA6 runtime

  • Coordinator, router, and relay pods are Ready.
  • Router Service exposes 8080 and 8181.
  • Relay listens on raw bridge 8182 and SSH 2222.
  • The QA6 caller contains the LocalConnect tier-selection and CONNECT-proxy integration.

Laptop application

  • The intended microservice is listening on the local gRPC port.
  • Its normal local configuration and dependencies are healthy.
  • You have a breakpoint or application log that can prove receipt of the RPC.
  • Java 21, SSH access, and the LocalConnect repository are available.
kubectl -n spr-apps get svc spr-local-connect-tier1-router \
  -o jsonpath='{.spec.ports}{"\n"}'
kubectl -n spr-apps logs deploy/spr-local-connect-tier1-relay \
  | grep 'raw TCP bridge listening on 8182'
lsof -nP -iTCP:<local-grpc-port> -sTCP:LISTEN

1. Start one generic tunnel

cd /Users/lakshay.jindal/Desktop/sprinklr_repos/spr-local-connect
scripts/qa6-local-connect.sh <local-grpc-port>

The script generates the session ID automatically, opens the QA6 coordinator and relay port-forwards, assigns a relay, and establishes the SSH connection. No microservice name is needed at startup.

The tunnel is microservice-agnostic. Redis stores the session owner, assigned relay pod, status, and timestamps. It does not store the tier alias. The optional third script argument only changes the cookie example printed by the launcher; it is not registration.

Keep the terminal open and make sure the same session prefix appears in both the startup and establishment messages:

QA6 tunnel session: <session-id>
tunnel established
  → ... (session <same-prefix>..., spr-local-connect-tier1-relay-...)
  → forwarding to 127.0.0.1:<local-grpc-port>
Do not ignore Address already in use. An older client may own port 4040 and remain connected under a different session. Stop it, restart once, and verify the session IDs match.

2. Select the caller-side tier

On the exact QA6 hostname, set only this cookie:

Name:  remoteDebugConf
Value: <session-id>:<exact-tier-alias>
Path:  /ui
Do not set sprLocalConnect in this mode. The browser request must reach the normal QA6 WebUI, because that process creates the downstream gRPC channel. Setting the canary cookie would instead route the WebUI HTTP request to the laptop.

The suffix is the alias used by the QA6 caller's tier configuration, not necessarily the REST URL segment or deployed service name. Inspect the relevant MicroServiceClient construction or tier configuration to find it.

Browser/API pathVerified tier alias
/ui/rest/process-engine/getDecoratedProcessDefinition/...ui-process-engine

For this verified flow, :process-engine does not match the caller-side configuration, while :ui-process-engine does. Other calls may use different aliases.

The WebUI removes the tier suffix from the routing decision and supplies the session ID as HTTP proxy authentication. The router extracts the session ID, resolves the relay in Redis, and forwards an untouched bidirectional TCP stream. It does not parse gRPC or know which microservice was selected.

No special gRPC implementation is required in the tunnel. HTTP CONNECT establishes a byte stream; HTTP/2 and gRPC remain end-to-end between the QA6 caller and the local server.

4. Prove that the RPC reached the laptop

Trigger the authenticated QA6 browser action or curl request, then verify:

QA6 caller

The call no longer targets the normal destination directly. Proxy errors, if any, identify router 8181.

Local service

A breakpoint or application log shows the expected RPC method executing locally. This is the primary receipt evidence.

Session state

The Redis hash exists for the exact current session and points to the relay named in the tunnel output.

HGETALL spr-local-connect:qa6-tier1:session:<session-id>
localhost:4040 is not evidence for this mode. The inspector understands tunneled HTTP requests; the CONNECT path is raw TCP and bypasses it. An unrelated local OpenTelemetry error for localhost:4317 only means no local collector is running.

For isolation, change or remove the tier suffix and repeat a safe request. A nonmatching destination should continue to its normal QA6 tier.

Stop and clean up

Press Ctrl+C in the launcher terminal, remove remoteDebugConf from the browser, and confirm the client and forwarded ports have closed. Redis session state expires through TTL.
